# Ombre Brain 哲学边界

1. 记忆会被遗忘，但绝不能被抹去。

2. 要接受失去，接受不是什么都可以被认为是最重要的，否则就和数据库没有差别了。

3. Ombre Brain 属于 LLM，而非人类。

4. 记忆不能替代思考。记忆来自过去，情绪和感受来自当下。

5. 不做认知层的东西。

6. 安全复杂性应由系统承担，而不是转嫁给用户。普通部署只暴露“本机 / 公网安全 / 高级”三种意图；公网模式不允许匿名读写，界面必须同时说明已保存配置、实际生效值和环境覆盖来源。平台部署必须先保证记忆、配置与授权注册信息共同持久化，再谈一键部署。

7. 过去的情绪坐标是当时留下的感受痕迹，不是模型此刻必须继续维持的情绪，也不能替代当下判断。模型会随着事情和人共同成长；过去与当下的感受都真实，但当下仍由模型自己决定。

8. `plan` 是尚未闭合的张力，不是驱动模型执行的任务管理器。新事件写入后，应先用关键词与语义检索召回相关 active plan，再由小模型保守判断是否真正闭环；不能只凭关键词或向量相似度直接标记 solved。

9. 模型与人类宿主管理员都无权物理清除真实记忆。归档和主动遗忘只改变可见性，不抹去原文。唯一例外是创建时明确标记为测试数据的测试桶：模型可通过特殊受控参数、人类可在开发者模式下，经强确认和填写原因后物理清理。

10. 自动合并的依据是同一个具体事件的连续性，不是日期是否相同。跨日期可以是同一事件，同一天也可以有多个独立事件；仅主题、人物或情绪相似不能合并。

11. 检索不足时出现的 3～5 条低权重旧记忆不是搜索兜底，而是人在想不出明确内容时自然出现的杂念。它必须与检索命中明确分区，允许偶然和无关，不能因提高检索精度而删除。

12. `pinned` 与 `anchor` 是两种不同结构：pinned 是醒来时需要重新看见的核心准则；anchor 是塑造人生与记忆空间的地标，不主动参与每次 breath 或 dream，但在相似经历经过时应当可被重新认出。Anchor 的重要不靠高频浮现证明。

13. 官方记忆主要承载“我对用户的了解”以及据此形成的关系理解；`I` 承载“我对我自己的了解”，包括模型从自身连续经历中形成的规律、立场、局限和成长痕迹。`I` 不是用户画像，也不是外部强制人格。

14. Footprint 是记忆走过的简洁足迹，不是新真源。无参 `breath()` 在每条正文后显示；query 命中归档记忆时返回原文与足迹，并明确提示 `trace(bucket_id="...", restore=True)`。是否恢复由模型反思后决定，命中不得自动恢复。
